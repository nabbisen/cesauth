# RFC 082 — Tenant admin デザイントークン統合

**Status**: Proposed  
**Tier**: P2  
**Size**: Medium  
**Target**: v0.63.0  
**Phase**: UI consistency  
**Refs**: G14 (gap analysis)

## Problem

- End-user `templates.rs::BASE_CSS` は `--success/--warning/--danger/--info` の
  CSS variables を使用 (WCAG AA + dark mode)
- `tenant_admin/frame.rs` および `tenancy_console/frame.rs` は**独自の色値を
  ハードコード** (`.badge.ok = #d4edda`, `.badge.warn = #fff3cd`, etc.)
- PDF は status token の統一を要求 (danger / warning / success / info)

これは「色の意味が画面間で微妙にズレる」コスト + 「dark mode 対応漏れ」を生む。

## Goal

すべての frame で共通の CSS variables (`--success` etc.) を使うようにする。

## Design

1. CSS variables を `crates/ui/src/design_tokens.css` (新規) として一箇所に集約。
2. 全 frame は `include_str!` で同じ token block を埋め込む。
3. `.badge.ok / .badge.warn / .badge.critical` を `.badge--success / --warning / --danger`
   に統一する (admin frame の既存 badge クラスを移行)。
4. 既存の `.scope-system / .scope-tenant` 色値も token 化。

## Implementation steps

1. `design_tokens.css` 新規 (CSS variables のみ)。
2. 3 frame の inline CSS から token block を `include_str!` に置換。
3. badge class 移行 (互換のため両方を alias で残し、admin pages を順次更新)。
4. visual regression tests (HTML 文字列で class 存在チェック)。

## Acceptance

- [ ] 3 frame で `--success` 等の token が定義されている
- [ ] `prefers-color-scheme: dark` で dark mode override が動く
- [ ] admin / tenant_admin / tenancy_console で同じ色 token が参照されている
- [ ] 全テストスイート green

# RFC 083 — Config preview に Rollback hint を追加

**Status**: Proposed  
**Tier**: P2  
**Size**: Small  
**Target**: v0.62.x  
**Phase**: Operations UX  
**Refs**: G9 (gap analysis), PDF "Config preview / apply: Pending change / Impact / Rollback hint / Apply with audit"

## Problem

`admin/preview.rs` は diff + impact severity を表示するが、**rollback 手順**を
画面に明示しない。PDF が要求する 4 要素 (Pending change / Impact / Rollback /
Apply) のうち Rollback が欠けている。

## Goal

各 preview ページに **"Rollback hint"** ブロックを追加し、apply 後に元に戻す
手順を明示する。

## Design

`ImpactStatement` に `rollback_hint: Option<String>` を追加。bucket-safety,
threshold, LOG_LEVEL 等の各 config 種別ごとに rollback hint を埋める。

例:
- **LOG_LEVEL**: "wrangler secret put LOG_LEVEL <previous-value> でロールバックできます"
- **bucket-safety**: "Operations 権限で再度 preview → apply してください"

`crates/ui/src/admin/preview.rs` で:

```html
<section class="rollback-hint" aria-labelledby="rollback-heading">
  <h3 id="rollback-heading">Rollback</h3>
  <p>{rollback_hint}</p>
</section>
```

## Implementation steps

1. `ImpactStatement.rollback_hint: Option<String>` 追加。
2. 各 config preview builder で rollback hint を埋める。
3. preview.rs UI で section render。
4. MessageKey `PreviewRollbackHeading` 追加 (JA admin only)。

## Acceptance

- [ ] LOG_LEVEL preview の HTML に rollback hint が含まれる
- [ ] bucket-safety preview の HTML に rollback hint が含まれる
- [ ] rollback hint なしの impact は section を出さない (Optional)
- [ ] 全テストスイート green

# RFC 084 — Sessions ページ drift repair note

**Status**: Proposed  
**Tier**: P2  
**Size**: Small  
**Target**: v0.62.x  
**Phase**: Self-service UX  
**Refs**: G2 (gap analysis), PDF Sessions panel: "Drift repair note"

## Problem

PDF Sessions panel は "Drift repair note" を要素として含める。これは
**D1-DO session-index drift detection** (ADR-012 §Q1) の結果をユーザーに
**さりげなく**表示することを意図している。

具体的には: D1 session index と DO session state がずれた場合 (drift)、
ユーザーが見ているリストが古い可能性を伝える。

## Goal

`/me/security/sessions` の下部に **muted な inline note** を追加:

> "セッション情報は数分の遅延がある場合があります。最新の状態は次回ログイン時に
> 反映されます。"

不要に大きく表示しない (footnote 的)。

## Design

### Detection

drift は cron で audit のみされており、ユーザー単位で「あなたのアカウントは
今 drift しています」という情報を持っているわけではない。本 RFC では**常に表示**
する general note とする。

ただし将来的に user 単位 drift がわかる仕組みができたら、表示有無を切り替える
拡張ポイントを残す:

```rust
pub fn sessions_page_for(
    items: &[SessionListItem],
    csrf_token: &str,
    flash_html: &str,
    locale: Locale,
    // 将来用:
    // drift_detected_for_user: bool,
) -> String { ... }
```

本 RFC では note は常時表示。

### MessageKey

```rust
SessionsDriftNote, // "セッション情報は数分の遅延がある場合があります" / "Session info may be delayed by a few minutes"
```

### HTML

```html
<p class="muted" role="note">
  {drift_note}
</p>
```

## Implementation steps

1. `MessageKey::SessionsDriftNote` 追加 + JA/EN 翻訳。
2. `sessions_page_for` 末尾に note を追加。
3. Test 追加。

## Acceptance

- [ ] /me/security/sessions HTML に drift note が含まれる
- [ ] JA / EN 両方で正しく表示
- [ ] 全テストスイート green
